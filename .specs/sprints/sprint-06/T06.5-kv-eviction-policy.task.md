# T06.5 - Define kv eviction policy behavior

## Context

The runtime has reuse and tiering signals, but it still lacks an explicit
policy contract for when and how older KV state should be demoted or evicted.

## Acceptance criteria

- [ ] Eviction policy is defined in code and/or architecture docs
- [ ] Page hotness and usage frequency influence the chosen policy
- [ ] Tests or simulations capture at least one eviction scenario

## Validation

- docs consistency
- build/lint/unit
- native contract runner when applicable
