# T01.6 - Plan CI and DoD transition from starter to runtime

## Context

Current workflows validate the starter package, not the Apple runtime. The planning must show how that transition will happen without pretending it already happened.

## Acceptance criteria

- [x] Current CI reality is documented honestly
- [x] Planned runtime CI path is documented
- [x] Release docs do not assume missing automation already exists

## Evidence

- `.specs/workflow/WORKFLOW.md` section 5 ("CI reality today") lists every workflow currently in `.github/workflows/` and what it enforces.
- `.specs/workflow/WORKFLOW.md` section 6 ("Next CI transition") describes remaining native gates without claiming they exist.
- `.specs/workflow/RELEASE.md` section 6 explicitly states that runtime release automation is planned and `.github/workflows/release.yml` does not exist yet.
- Workflows on disk: `.github/workflows/{ci.yml,dod.yml,scaffold-self-check.yml}` match the doc claims.

## Validation

- workflow docs consistent with actual repo files
