# T01.1 - Create root runtime scaffold and build contract

## Context

Sprint 01 starts the transition from starter/bootstrap to runtime implementation. The repo needs a concrete root layout and planned build contract before any adapter or backend code can land safely.

## Acceptance criteria

- [x] Root runtime layout is defined and reflected in docs
- [x] Planned build entrypoints are explicit
- [x] Scope separates "planned now" from "not yet present"

## Evidence

- Runtime tree exists under `runtime/` (`core`, `adapters`, `kv`, `moe`, `metal`, `mlx`, `neon`, `ane`, `memory`, `cache`, `speculative`, `tuning`, `telemetry`, `benchmarks`).
- `runtime/README.md` documents what exists now, what the scaffold already proves, and what is still missing.
- Build entrypoints in `CMakeLists.txt`, `runtime/CMakeLists.txt`, `apps/CMakeLists.txt`.
- `.specs/architecture/DESIGN.md` section 6 and `.specs/architecture/PATTERNS.md` section 2 mirror the runtime tree.
- `.specs/workflow/WORKFLOW.md` section 1 explicitly marks planned vs present scope.

## Validation

- docs consistency
- no broken references to generic app structure
