# T01.3 - Define adapter contract and runtime boundaries

## Context

Dense, MoE, and low-memory families need one shared runtime core. This task locks the boundary contract before implementation starts.

## Acceptance criteria

- [x] Runtime boundaries are named and stable
- [x] Adapter responsibilities are separated from backend responsibilities
- [x] Architecture docs stop using generic web-app concepts

## Evidence

- `.specs/architecture/ADR-002-runtime-boundaries-and-adapter-contract.md` locks the boundary set and adapter contract.
- `.specs/architecture/DESIGN.md` sections 2 and 4 describe boundaries and runtime contracts (`RuntimeSession`, `IUS4V6Adapter`, `BackendExecutor`, `MemoryGovernor`, `CorrectnessGuard`).
- `.specs/architecture/PATTERNS.md` sections 3, 4, 6, and 8 separate adapter responsibilities from backend responsibilities.
- `.specs/product/DOMAIN.md` lists adapter/backend entities; web-app vocabulary is gone.
